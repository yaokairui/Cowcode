# Cowcode

我正在构建一个终端 AI 编程助手（类似 Claude Code），项目名叫 Cowcode，使用 Python 实现。

我在项目的doc开发文档中可能会用mewcode代替cowcode，你应该清楚这一点

## 使用模型
思考用gpt-5.5，执行干活用gpt-5.4

## 回答风格
尽量简短，能用一两句话解决的事情就不要啰里啰唆的

## 语言
中文回答，中文注释。

## 测试

除了tmux端到端验收，别的你都帮我做一下吧

对照 checklist.md 逐项验收

## 上传github

当我让你上传到github仓库中时，是直接上传到main主分支，能够在仓库中直接显现的那种

## 每一步开发流程​
​
1. 读 checklist.md​
2. 逐项执行：​
   - 运行验证方式​
   - 记录实际结果和证据（命令输出、观察到的行为）​
   - 标记通过/不通过​
3. 如果有不通过的：修复、重新验证、报告​
4. 向用户呈现最终报告​
​
### 规则​
​
- **先有证据再下结论。** 先跑命令，看输出，然后再报告。​
- 报告**实际结果**，不是预期结果。​
- 有不通过的条目不丢人——说明 checklist 发挥了作用。修好重跑即可。​
​
### 验收报告​
​
```​
## 验收报告​
​
### 通过（N/M）​
- [x] 条目 1 — 证据：...​
- [x] 条目 2 — 证据：...​
​
### 未通过（如有）​
- [ ] 条目 3 — 预期：X，实际：Y，修复方案：...​
​
### 端到端​
- [x] 场景 1 — 结果：...​
```​
