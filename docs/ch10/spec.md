
# slash命令体系 Spec## 背景

当前 MewCode TUI 的输入框是单一入口：所有回车提交都作为 user message 送进 Agent 走 LLM。代码里已有一个最小的斜杠分发器，支持 5 条命令（/exit、/plan、/do、/compact、/resume），但存在几处明显的工程缺口：

- 命令注册表只有"名字 → handler"两栏，没有描述、别名、执行类型等元数据
- 没有启动期冲突检测，名字撞了会在运行时静默失效
- 没有 /help；可用命令清单硬编码在两处（未知命令提示文本 + 启动引导文本），互不同步会随版本漂移
- 没有自动补全，用户必须凭记忆敲完整命令
- handler 函数签名直接接收 TUI App 引用，没有 UI 抽象层，所有命令实现紧耦合在 TUI 包内
- 缺少"纯本地查询"类命令（看状态/记忆/权限/会话/帮助），这些非对话操作目前都要让 LLM 去做，浪费 token、行为不确定

## 目标- **G1**：把现有 5 条命令收编进新的注册中心；新增 7 条常用本地操作命令，共 12 条内置命令
- **G2**：让常用本地操作绕过 LLM——响应即时、不消耗 token、行为完全可预测
- **G3**：提供一层 UI 控制抽象，handler 不再直接持有 TUI App 引用,为后续扩展(Skill 系统等)解耦
- **G4**：注册中心驱动 /help 与未命中提示,单一信源;启动期检测名字与别名冲突直接立即终止启动并报错
- **G5**：在输入 `/` 后实时显示候选命令补全菜单,覆盖上/下/Tab/回车/ESC 全套键位

## 功能需求### 命令注册与分发- **F1**：提供一个内置命令注册中心,存储每条命令的名字、别名集合、一句描述、执行类型、处理函数;元数据形态对所有命令统一
- **F2**：注册中心在进程启动期对名字与别名做冲突检测,发现任一名字或别名重复立即终止启动并报错,不进入运行时
- **F3**：用户在输入框提交内容时,分发器先判断是否以 `/` 开头;以 `/` 开头者绕过 LLM 由本地分发,非 `/` 开头者送给 Agent
- **F4**：命令名匹配大小写不敏感
- **F5**：空输入与仅含空白字符的输入被早返回,不参与分发
- **F6**：未命中任何已注册命令名/别名时,向用户输出引导使用 /help 的提示
- **F7**：分发器对参数尾巴不做任何解析;本期所有内置命令均为零参数,任何以 / 开头但携带空白外尾随字符的输入按未命中处理

### 命令执行类型- **F8**：每条命令必须显式声明属于以下三类之一——纯本地、影响界面、提示词
- **F9**：纯本地类命令只向用户输出信息,不修改对话历史、不改变界面运行模式、不消耗 token
- **F10**：影响界面类命令可以修改运行时模式、清空或替换会话、退出进程、触发会话恢复,但不向对话历史追加 user 消息
- **F11**：提示词类命令向对话历史追加一条用户消息(含命令产生的固定文本),随后立即触发一次 LLM 回合;该消息按和真实用户消息相同的方式被持久化、对 LLM 可见

### 12 条内置命令- **F12**：内置 /exit,影响界面类,关闭 TUI 进程
- **F13**：内置 /plan,影响界面类,切换运行时权限模式到计划模式
- **F14**：内置 /do,提示词类——先把运行时权限模式切回默认模式,然后向对话注入现有的执行指令文本,并立即触发回合;外部行为与本 spec 实施前一致
- **F15**：内置 /compact,影响界面类,手动触发上下文压缩
- **F16**：内置 /resume,影响界面类,打开历史会话列表,选中后从该会话最后一个 compact 标记之后恢复;沿用 ch09 的所有约束
- **F17**：内置 /clear,影响界面类——关闭当前会话存档文件、开新会话存档文件、清空当前内存中的消息列表、把会话累计 token 计数与回合数归零;执行后向用户输出一条 notice,提示已结束当前会话并开启新会话
- **F18**：内置 /help,纯本地类,按命令名字典序排序输出每条已注册命令的"名字 + 一句描述"两列对齐列表
- **F19**：内置 /status,纯本地类,输出当前权限模式、累计 token 输入/输出、可用工具数量、已加载记忆条目数、当前模型名、当前工作目录六项信息
- **F20**：内置 /memory,纯本地类,输出当前已加载的项目层与用户层记忆文件名列表(只列文件名,不展开内容)
- **F21**：内置 /permission,纯本地类,输出当前权限模式的字符串名称(与 /status 中相同的字符串形式)
- **F22**：内置 /session,纯本地类,输出当前会话的标识信息——至少包含当前会话存档文件路径与 session 标识
- **F23**：内置 /review,提示词类,向对话注入一条固定文本的"代码审查请求"消息,并立即触发回合;不读 git diff、不收集任何外部上下文

### 自动补全菜单- **F24**：用户在输入框中第一个字符是 `/` 时,菜单激活并实时显示候选;用户继续输入,菜单按当前输入对所有已注册命令名做前缀匹配过滤
- **F25**：菜单不参与别名匹配、不参与描述匹配;仅按命令名前缀过滤
- **F26**：当前输入不再以 `/` 开头时(包括删空),菜单立即关闭
- **F27**：菜单每条候选显示"命令名 + 一句描述"两列对齐
- **F28**：菜单不显示标记为 hidden 的命令(机制保留;本期 12 条命令均不为 hidden);Hidden=True 的命令既不出现在 /help 输出中,也不出现在补全菜单中,但 dispatcher 仍能命中(为未来 Skill 系统预留)
- **F29**：菜单激活期间,上/下方向键在候选间切换高亮项
- **F30**：菜单激活期间,回车与 Tab 直接执行当前高亮的命令;执行后菜单关闭、输入框清空
- **F31**：菜单激活期间,ESC 关闭菜单;输入框内容保持不变,光标回到输入框
- **F32**：菜单关闭时,所有键位回到输入区默认行为
- **F32a**：输入包含换行符时菜单不激活;激活后输入变成多行时菜单立即关闭
- **F32b**：零匹配时菜单仅显示"无匹配"提示行;此时按回车走未命中提示分支、按 Tab/ESC 仅关闭菜单
- **F32c**：不支持鼠标点击候选项(本期 TUI 无 mouse handler)

### UI 抽象层- **F33**：命令处理函数不直接持有 TUI 渲染框架的类型;handler 通过一个抽象接口访问 UI 能力——向用户打印消息、注入并发送 user 消息、切换权限模式、查询累计用量、查询当前模式/模型/目录/记忆文件名清单、关闭进程、打开会话列表、触发压缩、清空并新建会话等
- **F34**：抽象接口对实现要求最小化——只暴露命令实际需要的方法,不把 TUI App 内部属性直接外露
- **F35**：注册期内置命令时,handler 函数签名仅依赖该抽象接口,不依赖具体 TUI 实现

## 非功能需求- **N1**：所有纯本地命令在用户回车后必须在主线程(asyncio event loop)上完成输出,无可观察延迟
- **N2**：影响界面类命令的状态变更对用户立即可见(模式切换的彩色徽章在下一帧渲染中生效)
- **N3**：提示词类命令的注入消息对 LLM 不可与真实用户消息区分;具体表现为走相同的对话历史追加、相同的会话存档持久化路径
- **N3a**：影响界面类(KindUI)与提示词类(KindPrompt)命令仅在 idle 状态可执行,非 idle 时拒绝并提示"请等待当前任务完成";纯本地类(KindLocal)在任何状态下都可执行(不改 App、只读查询)
- **N4**：注册期冲突检测的报错信息必须包含冲突的具体名字或别名,便于排查
- **N5**：自动补全菜单内容渲染高度上限为固定行数;候选超过时菜单内部可滚动
- **N6**：自动补全菜单位置紧贴输入框下方、状态栏上方;不占用现有状态栏渲染区
- **N7**：/help 与未命中命令提示的命令列表必须来自同一注册中心查询,不允许任何硬编码字符串;ReadyHint 文案中允许且仅允许出现 /help 作为入口引导,其他 11 个命令名不得在 mewcode.prompt 模块内任何处出现
- **N8**：/status 的 6 个字段渲染顺序与 F19 列举顺序固定一致;每行 key 与 value 之间按列对齐
- **N9**：/clear 写入新会话存档文件后,/resume 列表能看到旧会话作为可恢复条目
- **N10**：收编后,现有 5 条命令的外部行为与本 spec 实施前一致(包括 /resume 仅在 idle 状态可用、/compact 通过同一事件流推送压缩进度等)
- **N11**：状态栏的左右两侧字段、宽度、高度、模式可视标记渲染保持现状不动
- **N12**：/exit 在通知 TUI 退出前必须取消 TUI 持有的主 asyncio cancel scope,让 Agent.run、memory.update_async、MCP 等后台任务收到 CancelledError;具体进程资源(会话存档 writer 等)由 TUI shutdown 钩子关闭

## 不做的事

- 不支持命令参数(架构上不预留参数 tokenizer)
- 不动状态栏现有的彩色 mode 徽章渲染
- 不修改 /resume、/compact、/plan、/do、/exit 的既有外部行为
- 不支持用户自定义命令、命令热加载、动态生成提示词
- 不引入命令级权限控制(留给 Skill 系统)
- /review 不读 git diff、不组织结构化审查输入、不调用任何外部命令
- /memory 不展开任何记忆条目内容、不提供编辑入口、不触发记忆重载
- /permission 不能修改权限模式、不能编辑持久化规则
- /session 不能切换或操作会话(操作类入口由 /resume、/clear 各自提供)
- /help 不分组、不显示别名、不支持详情子查询
- 不支持鼠标点击补全候选项

## 验收标准- **AC1**：键入 `/help` 回车,输出按字典序排列的 12 行内容,每行"命令名 + 一句描述"两列对齐
- **AC2**：键入未注册的 `/foobar` 回车,输出引导用户使用 /help 的提示,且不触发任何 LLM 调用
- **AC3**：键入 `/Help`(大小写混合)回车,行为与 /help 一致
- **AC4**：键入 `/status` 回车,输出六行 key:value,渲染顺序与 F19 列举顺序一致
- **AC5**：键入 `/memory` 回车,输出当前已加载的项目层与用户层记忆文件名列表
- **AC6**：键入 `/permission` 回车,输出当前权限模式名称
- **AC7**：键入 `/session` 回车,输出当前会话的会话存档路径与 session 标识
- **AC8**：键入 `/clear` 回车,对话区域清空、累计 token 计数归零;再触发一次 LLM 回合后,在 /resume 列表里能看到上一次的旧会话条目
- **AC9**：键入 `/review` 回车,状态栏立即进入流式状态、AI 开始回复;会话存档中新增一条 user 角色的消息,文本含审查相关关键字
- **AC10**：现有 /exit、/plan、/do、/compact、/resume 的外部行为与本 spec 实施前完全一致
- **AC11**：在输入框首个字符输入 `/` 时,菜单立即激活并显示全部 12 条候选
- **AC12**：继续输入 `s`(输入框为 `/s`),菜单仅显示以 /s 开头的候选
- **AC13**：菜单激活时按上下方向键,高亮在候选间切换;按回车,所选命令立即执行
- **AC14**：菜单激活时按 ESC,菜单消失,输入框光标可继续编辑 `/s`
- **AC15**：把输入框 `/` 删除后,菜单立即消失
- **AC16**：在源码中给某条已注册命令名再注册一个同名命令,启动 mewcode,进程立即终止启动并报错并打印冲突名字
- **AC17**：在 tmux 内启动 mewcode,依次执行 /help → /status → /memory → /permission → /session → /review,全程不卡顿、所有输出按预期渲染、无错误日志(具体步骤见 checklist 端到端场景)
